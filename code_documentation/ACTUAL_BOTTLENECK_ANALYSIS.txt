THE REAL BOTTLENECK: INSTANCED SPHERE RENDERING
================================================================================

SPHERE GEOMETRY COMPLEXITY:
---------------------------
- Sphere generated with 3 iterations (renderer.js line 193)
- Starting shape: Icosahedron (12 vertices, 20 triangular faces)
- Each iteration subdivides each face into 4 faces
- After 3 iterations: 20 × 4³ = 20 × 64 = 1,280 faces
- Each face = 2 triangles = 6 indices
- Total indices per sphere: ~7,680 indices

RENDERING PASSES THAT DRAW SPHERES:
------------------------------------
1. SPHERE RENDERING PASS (renderer.js line 398)
   → Draws all particles as spheres
   → Outputs: normal.x, normal.y, speed, depth

2. AMBIENT OCCLUSION PASS (renderer.js line 449)
   → Draws all particles as spheres again
   → Calculates occlusion from nearby particles

3. SHADOW MAP PASS (renderer.js line 494)
   → Draws all particles as spheres from light's perspective
   → Depth only, but still full geometry

TOTAL GEOMETRY PER FRAME:
-------------------------
With 1,219,450 particles:

Per Pass:
- 1,219,450 particles × 7,680 indices = 9,365,376,000 indices
- That's 9.3 BILLION indices per pass!

All 3 Passes Combined:
- 9.3 billion × 3 passes = 27.9 BILLION indices per frame
- At 19 FPS: 27.9 billion × 19 = 530 BILLION indices per second!
