================================================================================
COMPOSITE SHADER PERFORMANCE ANALYSIS
================================================================================

BASELINE ASSUMPTIONS:
---------------------
- Canvas Resolution: 1920 x 1080 pixels (default from index.html)
- Total Pixels per Frame: 2,073,600 pixels
- Shadow Map Resolution: 256 x 256 (SHADOW_MAP_WIDTH/HEIGHT)
- Target Frame Time: 16.67ms (60 FPS)
- PCF Shadow Filtering: 5x5 samples (PCF_WIDTH = 2)

================================================================================
COMPOSITE SHADER COST BREAKDOWN (PER PIXEL):
================================================================================

1. CPU-SIDE COSTS (Per Frame):
   ----------------------------
   - Matrix Inversion (inverseViewMatrix): ~0.1ms
     → 4x4 matrix inversion using Gauss-Jordan elimination
     → ~100 floating point operations
   
   - WebGL State Setup: ~0.05ms
     → Framebuffer binding, viewport, program binding
     → Uniform setting (12 uniforms)
   
   TOTAL CPU COST: ~0.15ms per frame

2. GPU-SIDE COSTS (Per Pixel):
   ----------------------------
   
   A. TEXTURE SAMPLES:
      - u_renderingTexture: 1 sample (RGBA, float)
      - u_occlusionTexture: 1 sample (RGBA, unsigned byte)
      - u_colorRamp: 1 sample (RGBA, unsigned byte)
      - u_shadowDepthTexture: 25 samples (PCF 5x5 filter)
        → PCF_WIDTH = 2 means -2 to +2 = 5 samples per axis
        → 5 × 5 = 25 shadow samples per pixel
      
      TOTAL TEXTURE SAMPLES: 28 per pixel
   
   B. ARITHMETIC OPERATIONS:
      - Normal reconstruction: sqrt(1 - x² - y²) = ~10 ops
      - View ray calculation: 3 ops
      - View space position: 1 op
      - World space position: 4×4 matrix × vec4 = 16 ops
      - Light space position: 4×4 matrix × vec4 = 16 ops
      - Light space coordinate transform: 4 ops
      - Shadow calculation loop: 25 iterations × 2 ops = 50 ops
      - Color ramp lookup: 5 ops
      - Lighting (ambient + direct): 3 ops
      - Glow effect (if enabled): 8 ops
      - Sparkle effect (if enabled): 5 ops
      - Bloom effect (if enabled): 10 ops
      - Vignette effect: 5 ops
      - Final color clamping: 3 ops
      
      TOTAL ARITHMETIC OPS: ~140 operations per pixel
      (More if effects enabled: +23 ops for glow+sparkle+bloom)
   
   C. CONDITIONAL BRANCHES:
      - Speed check (speed >= 0.0): 1 branch
      - Glow enabled check: 1 branch
      - Sparkle enabled check: 1 branch
      - Bloom enabled check: 1 branch
      
      TOTAL BRANCHES: 4 per pixel

================================================================================
PERFORMANCE CALCULATION:
================================================================================

PER PIXEL COST:
---------------
- 28 texture samples × ~0.5ns = 14ns
- 140 arithmetic ops × ~0.1ns = 14ns
- 4 branches × ~1ns = 4ns
- Total per pixel: ~32ns

PER FRAME COST (1920×1080):
---------------------------
- GPU: 2,073,600 pixels × 32ns = 66.4ms
- CPU: 0.15ms
- TOTAL: ~66.6ms per frame

This would give: 1000ms / 66.6ms = ~15 FPS

BUT WAIT - This is theoretical maximum. In practice:
- GPU pipelines are highly parallel
- Modern GPUs can process thousands of pixels simultaneously
- Texture cache reduces memory access time
- Branch divergence is handled efficiently

REALISTIC PERFORMANCE:
----------------------
Based on typical GPU performance:
- Modern GPU (GTX 1060 or better): ~2-5ms per frame
- Mid-range GPU: ~5-10ms per frame
- Lower-end GPU: ~10-20ms per frame

================================================================================
WHEN COMPOSITE SHADER IS DISABLED:
================================================================================

PER PIXEL COST:
---------------
- 1 texture sample (renderingTexture) × ~0.5ns = 0.5ns
- FXAA shader: ~5 texture samples × 0.5ns = 2.5ns
- FXAA arithmetic: ~20 ops × 0.1ns = 2ns
- Total per pixel: ~5ns

PER FRAME COST (1920×1080):
---------------------------
- GPU: 2,073,600 pixels × 5ns = 10.4ms
- CPU: 0.05ms (no matrix inversion)
- TOTAL: ~10.5ms per frame

This would give: 1000ms / 10.5ms = ~95 FPS

REALISTIC PERFORMANCE:
----------------------
- Modern GPU: ~0.5-1ms per frame
- Mid-range GPU: ~1-2ms per frame
- Lower-end GPU: ~2-4ms per frame

================================================================================
PERFORMANCE IMPROVEMENT:
================================================================================

THEORETICAL IMPROVEMENT:
------------------------
- With Composite: ~66.6ms/frame = 15 FPS
- Without Composite: ~10.5ms/frame = 95 FPS
- Improvement: 6.3x faster (530% speedup)

REALISTIC IMPROVEMENT:
---------------------
- With Composite: ~2-20ms/frame (depending on GPU)
- Without Composite: ~0.5-4ms/frame (depending on GPU)
- Improvement: 4-5x faster (300-400% speedup)

FRAME TIME SAVINGS:
------------------
At 60 FPS target (16.67ms per frame):
- With Composite: Uses 2-20ms (12-120% of frame budget)
- Without Composite: Uses 0.5-4ms (3-24% of frame budget)
- Savings: 1.5-16ms per frame

This means:
- If you're GPU-bound and getting 30 FPS with composite,
  disabling it could get you 60+ FPS
- If you're getting 60 FPS with composite,
  disabling it could get you 120+ FPS (if monitor supports it)

================================================================================
BREAKDOWN BY OPERATION TYPE:
================================================================================

MOST EXPENSIVE OPERATIONS (when enabled):
-----------------------------------------
1. Shadow PCF (25 samples): ~50% of total cost
   → 25 texture samples per pixel
   → This is the biggest performance hit

2. Matrix Multiplications: ~20% of total cost
   → World space position calculation
   → Light space position calculation

3. Texture Samples (non-shadow): ~15% of total cost
   → Rendering, occlusion, color ramp textures

4. Post-Processing Effects: ~10% of total cost
   → Bloom, sparkle, glow, vignette
   → Only if enabled

5. Arithmetic Operations: ~5% of total cost
   → Vector math, color calculations

================================================================================
OPTIMIZATION OPPORTUNITIES:
================================================================================

If you wanted to optimize the composite shader:

1. REDUCE SHADOW QUALITY:
   - Change PCF_WIDTH from 2 to 1 (3×3 = 9 samples instead of 25)
   - Performance gain: ~40% faster
   - Visual quality: Slightly softer shadows

2. DISABLE SHADOWS ENTIRELY:
   - Skip shadow map sampling
   - Performance gain: ~50% faster
   - Visual quality: No shadows (flat lighting)

3. REDUCE RESOLUTION:
   - Render composite at half resolution (960×540)
   - Upscale with FXAA
   - Performance gain: ~75% faster
   - Visual quality: Slightly softer image

4. CONDITIONAL EFFECTS:
   - Only calculate effects if enabled
   - Current code already does this (good!)

================================================================================
CONCLUSION:
================================================================================

DISABLING COMPOSITE SHADER GIVES:
- 4-5x performance improvement (realistic)
- 6.3x performance improvement (theoretical maximum)
- Saves 1.5-16ms per frame (depending on GPU)
- Can turn 30 FPS → 60+ FPS (if GPU-bound)
- Can turn 60 FPS → 120+ FPS (if monitor supports)

THE BIGGEST COST:
- Shadow PCF filtering (25 samples per pixel) = 50% of cost
- This is why shadows are expensive in real-time rendering

RECOMMENDATION:
- Use composite shader toggle for:
  * Performance testing
  * Debugging rendering issues
  * Low-end hardware support
  * High FPS competitive scenarios
